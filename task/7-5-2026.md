Tính năng **Favorites/Wishlist** đã được triển khai hoàn chỉnh. Dưới đây là tổng kết:

## 📋 Schema Database

- **Thêm model `Favorite`** vào `packages/product-db/prisma/schema.prisma` với:
  - `userId` + `hotelId` (unique constraint đảm bảo mỗi user chỉ thích 1 hotel 1 lần)
  - Quan hệ `User` ↔ `Favorite` ↔ `Hotel`
  - Cascade delete khi user/hotel bị xóa
- **Đã chạy migration** thành công: `20260507053713_add_favorites`

## 🔌 API Routes (Next.js App Router)

1. **`/api/favorites`** — `GET` (lấy danh sách yêu thích phân trang) + `POST` (thêm vào yêu thích + ghi interaction cho AI)
2. **`/api/favorites/[hotelId]`** — `GET` (kiểm tra hotel có trong favorites không) + `DELETE` (xóa khỏi favorites)

## 🎣 Frontend Hooks & Context

- **`useFavorites`** (`hooks/useFavorites.ts`) — Hook quản lý state favorites với optimistic updates
- **`FavoritesProvider`** (`contexts/FavoritesContext.tsx`) — Context provider share state toàn app
- **`useFavoritesSafe`** — Hook an toàn khi dùng ngoài provider (trả defaults thay vì throw error)

## 🧩 Components Cập Nhật

- **`BtnLikeIcon`** — Đã kết nối API: nhận prop `hotelId`, gọi `toggleFavorite()` khi click, hiển thị đúng trạng thái từ DB
- **`StayCard`** — Truyền `hotelId={id}` cho `BtnLikeIcon`
- **`providers.tsx`** — Wrap `FavoritesProvider` trong root providers
