## 1. Tong quan he thong

### 1.1 Muc tieu

Xay dung he thong goi y khach san ho tro **dual-feedback architecture**:

- **Implicit Feedback** (hanh vi ngam): CLICK_BOOK_NOW, ADD_TO_WISHLIST, BOOK -> **Ranking** (Top-K recommendations)
- **Explicit Feedback** (danh gia ro): Rating 1-5 -> **Rating Prediction** (RMSE, MAE)

### 1.2 Kien truc

```
+-----------------------------------------------------------+
|                    DATA PIPELINE                           |
|  Synthetic Data Generator (5 controlled patterns)          |
|  +-- User-Hotel Segment Clustering                        |
|  +-- Rating by Segment Match + Noise                      |
|  +-- Realistic Review Behavior                            |
|  +-- Multi-signal Funnel (CLICK -> WISHLIST -> BOOK)      |
|  +-- Exploration Behavior (30%)                           |
|                                                           |
|  Output: 255 hotels, 200 users, 7,913 interactions,      |
|          1,526 reviews                                    |
+-----------------------------------------------------------+
                              |
                              v
+-----------------------------------------------------------+
|                   MODEL TRAINING                           |
|  evaluate.py -> Dual-Feedback CF Evaluation                |
|  train_svd.py -> SVD + GridSearchCV (24 configs x 3-fold) |
|  src/recommend.py -> Multi-Strategy Engine (5 strategies)  |
+-----------------------------------------------------------+
                              |
                              v
+-----------------------------------------------------------+
|                 PRODUCTION DEPLOYMENT                      |
|  FastAPI: /recommend/{user_id}?strategy=svd               |
|  Admin Dashboard: AI Management + Training Logs + Chatbox  |
|  Client: PersonalizedSection (BentoGrid 7 hotels)         |
|  Cron: APScheduler 3:00 AM daily auto-retrain             |
+-----------------------------------------------------------+
```

### 1.3 Bang phan cong danh gia

| He thong              | Input                        | Model                          | Metric                        | Muc tieu                  |
| --------------------- | ---------------------------- | ------------------------------ | ----------------------------- | ------------------------- |
| System A: Implicit CF | interactions.json            | User-User CF (signal-weighted) | Precision@5, Recall@5, NDCG@5 | Ranking — recommend Top-K |
| System B: Explicit CF | reviews.json (rating)        | User-User CF (rating-based)    | RMSE, MAE                     | Rating Prediction         |
| System C: SVD Model   | Merged (implicit + explicit) | SVD (Matrix Factorization)     | RMSE, MAE, 5-fold CV          | Production model          |

## 3. Ket qua danh gia Implicit CF (Ranking)

### 3.1 Cau hinh

```
Algorithm:      User-User Collaborative Filtering (Cosine Similarity)
Signal Weights: CLICK_BOOK_NOW=2.0, ADD_TO_WISHLIST=3.0, BOOK=5.0
K Neighbors:    5
K Recommendations: 5
Split:          Temporal (60% train / 20% val / 20% test)
```

### 3.2 Ket qua

| Metric          | CF Model   | Baseline (Popular) | Improvement |
| --------------- | ---------- | ------------------ | ----------- |
| **Precision@5** | **0.0330** | 0.0210             | **+57.14%** |
| **Recall@5**    | **0.0200** | 0.0153             | **+30.32%** |
| **NDCG@5**      | **0.0307** | 0.0221             | **+39.07%** |

### 3.3 Phan tich

- CF luon beat baseline o ca 3 metrics -> chung to user-hotel segment clustering hoat dong dung
- Recall improvement cao nhat -> CF tim duoc nhieu item relevant hon baseline
- NDCG improvement -> thu tu ranking cua CF tot hon baseline (items duoc xep cao hon)
- Absolute values thap (0.03-0.05) -> do sparsity 91.2% va dataset nho (200 users x 255 hotels)
- Trong thuc te, Precision@5 khoang 3-5% la binh thuong cho sparse implicit data. Gia tri tuyet doi thap nhung **relative improvement so voi baseline la dang ke** (39-57%).

### 3.4 Ma tran

## 4. Ket qua danh gia Explicit CF (Rating Prediction)

### 4.1 Cau hinh

```

Split:          Temporal (60% train / 20% val / 20% test)
```

### 4.2 Ket qua

| Metric   | CF Model | Baseline (User Mean) | Improvement           |
| -------- | -------- | -------------------- | --------------------- |
| **RMSE** | 1.2852   | 1.2330               | **-4.23%** (CF worse) |
| **MAE**  | 1.0116   | 0.9870               | **-2.49%** (CF worse) |

### 4.3 Phan tich nguyen nhan

**Tai sao Memory-based CF thua Baseline?**

```
Sparsity: 98.23% = chi 1.77% cells co rating
-> Moi user co trung binh ~7.6 ratings tren 255 hotels
-> Khi tinh user-user similarity, hau het cap user share 0-1 items
-> Cosine/Pearson correlation vo nghia
-> Baseline "user mean" (~3.8) da la du doan rat tot
```

**-> Day la HAN CHE CUA MEMORY-BASED CF (khong phai loi data). Can Matrix Factorization (SVD) de giai quyet.**

## 7. Multi-Strategy Recommendation Engine

### 7.1 5 Strategies

| Strategy      | Function                  | Algorithm                    | Use Case              |
| ------------- | ------------------------- | ---------------------------- | --------------------- |
| svd (default) | svd_recommend()           | SVD + Content hybrid (60/40) | Production, user cu   |
| user_cf       | user_based_cf_recommend() | Cosine K=10 neighbors        | So sanh, fallback     |
| item_cf       | item_based_cf_recommend() | Item-item cosine similarity  | "Khach san tuong tu"  |
| content       | content_recommend()       | Category/tag matching        | User moi (onboarding) |
| popular       | popular_recommend()       | reviewStar x reviewCount     | Fallback cuoi cung    |

### 7.2 Strategy Dispatcher

```python
# API Usage
GET /recommend/{user_id}?strategy=svd      # Default — production
GET /recommend/{user_id}?strategy=user_cf  # User-based CF
GET /recommend/{user_id}?strategy=item_cf  # Item-based CF
GET /recommend/{user_id}?strategy=content  # Content-based
GET /recommend/{user_id}?strategy=popular  # Popular fallback
GET /similar/{hotel_id}                     # Similar hotels
```

### 7.3 Loi ich cho luan van

- Hoi dong co the **demo truc tiep** su khac biet giua cac thuat toan bang cach doi ?strategy=
- Co **baseline comparison** ro rang (popular vs CF vs SVD)
- **Fallback chain** an toan: SVD -> Content -> Popular
